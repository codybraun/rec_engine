    Authentication

  • Post a username and password to "/token" to receive a token which can be used to validate all future requests.
  • The token can then be sent as an "Authorization" header with "Token " prepended. (Like "Authentication: Token YOUR_TOKEN_HERE")

    Users

  • Post to "/users" to create a new user, which is returned as a JSON object with included id.

    Products

  • Post to "/products" to create a new product, which is returned as a JSON object with included id.

    Activity

  • Post to user/:user_id/product/:product_id/activity to register a new user-product activity, representing a user's affinity for a product.
  • A "score" parameter reflects the strength or weakness of that relationship (which may be represented by a star rating, hours used, etc.)

    Recommendations

  • Get /user/:user_id/recs will return a list of the most recommended products
